JavaScript White Badge

Scope

Scope means lifespan of variables.
It determines accessibility/visibility of the variables inside the JavaScript code
There are three types of scope:

Global Scope - global
variable declared in the global execution context
accessible from anywhere within the code
Function Scope - local
variable declared inside the function
cannot be accessed or modified outside the function
Block Scope - local
variable declared inside the curly braces
cannot be accessed or modified outside the curly braces

Lexical Scope

set of rules the JS Engine looks up variable by its identifier
innerScope -> outerScope -> globalScope

Hoisting

JS engine compile code before it interprets it.
JS Engine scans entire code during compiler phase, and store variable as "undefined" in memory under it's specific scope (global, functional, block).
var/function declarations are hoisted and initialised as "undefiend"
let/const declarations are hoisted but stay uninitualised and throw ReferenceError. Before initialisation is called temporal dead zone.

RHS reference
console.log(a) - nothing is being assigned to variable a, instead we are retrieving the value of a
LHS reference
a = 2 - we don't care about the value, we want to find the variable as the target of the = 2 assignment operation

"strict mode" disallows global scope to create global variables on its own.

Errors

ReferenceError is Scope resolution-failure related
TypeError implies that Scope resolution was succesful, but tehre was illegal/impossible action attempted against the result
